> **项目代号**:反常识勇者(暂定,可改)

## 1. 核心 idea(一句话)

一个 RPG,玩家扮演被洗脑的魔王军封印师"勇者",**最强开局,越打越弱**,6 场 boss 战揭示"国王才是真凶"的真相,最终在国王战中觉醒、收回能力、击败国王。

---

## 2. 设计灵魂(最重要)

### 核心洞察:**失去 ≠ 变弱,失去 = trade-off + 觉醒的代价**

每次使用封印术:
- 失去战斗力(技能 / 装备 / 等级 / 珍贵道具)
- **同时获得**:
  - 解开下一关的钥匙(洞察力 / 谜题线索 / 特殊解法)
  - 削弱洗脑魔法 → 玩家逐渐恢复记忆

这意味着:
- **机制层面**:玩家用"战斗力"换"洞察力"
- **叙事层面**:主角在战斗中逐渐觉醒
- **形式即内容**:整条"递减曲线"本身就是叙事结构

### 核心循环(2026-06-26 重要更新)

```
[每场战斗前] 主角拥有 X 样东西(开局 4 样,过程中递减)
   ↓
[战斗开始] 玩家不知道哪样东西是关键
   ↓
[独特的 ACT 选项] 让玩家"发现"哪样东西是关键
   ↓
[玩家选择] 用关键东西战胜敌人
   ↓
[战后] 失去那件东西 + 获得洞察力(下一关解法钥匙 + 削弱洗脑魔法)
   ↓
[下一关] 用剩余的东西 + 上一关的洞察 → 战胜下一个敌人
```

**Undertale 的对应**:
- ACT 选项 → "发现关键"的机制
- 选对 ACT 触发反应 → 玩家"恍然大悟"
- 黄名 → "用关键东西战胜"的状态

### 双线塑造世界观

| 场景 | 线 1 | 线 2 |
|---|---|---|
| 战斗 | 主角 ↔ 魔族(boss) | (boss 攻击模式暗示真相) |
| 休息 | — | 主角 ↔ 小精灵 |

- **战斗对话**:揭示魔族不是坏人
- **休息对话**:友情 + 主角的零星记忆

两个角度同时推进,玩家获得的信息比单线丰富。

### 情感基调(多幕剧)

| 阶段 | 基调 | 触发 |
|---|---|---|
| 前期(开场 ~ 战斗 2) | 轻松搞怪 + 友情培养 | 小精灵的闲聊 + 战斗中的轻互动 |
| 中期(战斗 3 ~ 4) | 觉醒 + 态度转变 | 双线剧情逐渐揭露真相 |
| 后期(魔王 ~ 国王) | 悲壮 + 救赎 | 主角恢复记忆 + 真结局战斗 |

## 3. 完整故事线(剧情梗概)

```
[背景]
主角原本是魔王军的封印师,擅长"献出一样东西封印敌人"的招式。
国王想除掉魔族(因魔族拥有某种资源/力量)——但魔族难杀,只能封印。
国王抓来主角,洗脑成"勇者",派他封印四天王和魔王。

[旅途]
主角在旅途中遇到小精灵,结伴同行。
每场四天王战:
  ├ 用不同的"特殊解法"战胜敌人
  └ 战后被迫使用封印术 → 失去一样东西(战斗力) + 获得下一关的解法钥匙(洞察力)
每段休息:
  └ 主角和小精灵篝火夜话 → 友情 + 主角的零星记忆浮现

[真相浮现]
随着封印术的使用,洗脑魔法逐渐失效。
主角通过战斗对话(线 1)和休息对话(线 2),逐渐意识到:
  ├ 魔族不是坏人
  └ 国王才是真正的敌人

[魔王战]
魔王战(综合解法),主角几乎失去所有。
最后两个选择:
  ├ 失去生命 → 悲剧结局(封印魔王,主角也没了)
  └ 揭示真相,停止魔王战 → 真结局 → 转国王战

[国王战]
小精灵"拉出迷惘"(友情羁绊的高潮) → 主角完全恢复记忆。
主角收回失去的能力(技能 / 装备 / 等级 / 珍贵道具)。
用全盛状态击败国王 → 救赎结局。

[多结局]
- 救赎结局(真结局):不杀魔王,转国王战 → 收回能力 + 击败国王 → 救赎
- 悲剧结局:杀魔王 → 自己也没了 → 悲壮
```

## 5. 关键技术决策

| 项目 | 决策 | 备注 |
|---|---|---|
| 类型 | 回合制 RPG | 适合穿插对话和解密 |
| 技术栈 | **Godot 2D** | 开源轻量,可拓展到 3D |
| 战斗系统 | 回合制 + 6 场战斗各用不同解法 | ABCE 四种 + 综合 |
| 休息场景 | 纯对话界面(篝火夜话) | 不需要走动 |
| 战斗解法 | ABCE 四种混合 | 每场战斗不同 |
| 变弱触发 | 战后强制封印 | "魔族不能杀只能封印" |
| 双线塑造 | 战斗对话 + 休息对话 | 两个角度推进 |
| 失去内容 | 技能 / 装备 / 等级 / 珍贵道具 | 各对应一场战斗的"关键" |

### 战斗解法 4 种

- **A. Undertale 式**:行动选择 / 对话选项触发特殊剧情
- **B. Paper Mario 式**:特殊装备 / 状态让某些选项生效
- **C. 观察攻击模式**:回合中根据敌人前一回合行动选对应行动
- **E. 对话回合**:战斗中触发对话,选项影响战局

### 失去 vs 获得 trade-off(核心机制)

每次失去:
- 战斗力 ↓
- 洞察力 ↑(下一关解法钥匙)
- 洗脑魔法 ↓(主角恢复记忆)

### 多结局

- 悲剧结局:失去生命
- 救赎结局:觉醒 + 收回能力 + 击败国王

## 9. 关键风险与规避

| 风险 | 规避 |
|---|---|
| 6 场战斗战线太长,容易烂尾 | 分段做,每段验证机制 |
| 对话量大,容易写崩 | 先写大纲,后填细节;借助小精灵提示推动对话 |
| "失去 = trade-off"机制不成立 | 阶段 1 先验证,不成立就调 |
| 双线塑造信息混乱 | 信息分散在不同场景,玩家不会一次性被淹没 |
| 失去后战斗太弱,玩家卡关 | trade-off 给的洞察力要足以补偿 |
| 多次调整方向 | 每个阶段完成后做一次回顾,坚持推进 |
| 4 个四天王设计同质化 | 每个天王的"怕什么"要有完全不同的逻辑 |

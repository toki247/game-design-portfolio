# 大型怪物战斗 demo 学习计划 v1.2 - 日历版

> **基础**:v1.1 计划已锁定(1 只怪 + 3D + UE 5.4 LTS + 大凶豺龙)
> **本版新增**:每天具体任务 + 时长 + 用户/Mavis 任务分配
> **时间假设**:工作日 2h/天 + 周末 4h/天 = 18h/周
> **总工作量**:6 个月 × 4 周 × 18h = **~432 小时**
> **任务分配**:用户(设计/填答/试玩)40% / Mavis(代码/美术/文档)50% / 协作 10%

## 风险预案

### 风险 1:某周延迟
**症状**:某周产出物未完成
**应对**:
- 优先完成"Mavis 任务"占 60%(代码 / 美术)
- 用户任务可以延后 1 周(填答可以慢)
- 检查点未达 → 多花 1 周调优,**不跳到下一阶段**

### 风险 2:UE 学习曲线陡(W9-W12)
**症状**:W11/W12 卡在 Anim BP / BT
**应对**:
- 退回 Blueprint 简化版(主角逻辑用 BP)
- 看 UE 官方模板源码(Third Person / Lyra)
- 跳过 GAS / 网络同步 / Niagara 高级

### 风险 3:用户时间不足(加班/出差)
**症状**:工作日 2h 投入不了
**应对**:
- 周末集中补(一天 8h × 2 = 16h)
- 关键任务:用户填答(招式表/参数)不可压缩
- 次要任务:文档整理可以慢

### 风险 4:电脑配置不够
**症状**:UE 跑不动(< 30fps)
**应对**:
- 调低 UE 画质(1080p → 720p)
- 用更简单的占位资源
- 严重时退回 Godot 2D 方案

## 每日固定流程(模板)

每个工作日 / 周末日,按以下流程执行:

```
[5 min]  看今日任务(从本文档)
[60 min] 主要任务(占 50%)
[30 min] 次要任务(占 25%)
[15 min] 整理产出(占 12%)
[10 min] 明日准备(占 8%)
---
总时长:2h(工作日) / 4h(周末)
```

**主要任务**:本周核心推进(如"填 sheet 0 频率分析")
**次要任务**:辅助 / 校对 / 整理(如"Mavis 校对笔记")
**整理产出**:把今日产出写进文档 / git commit(工作日最后一版)
**明日准备**:写下明日任务清单(用 TodoWrite)

## 工作日历备份方案

如果某周彻底停摆(比如公司紧急加班),备选方案:

**方案 A:周末集中补**(推荐)
- 周六 8h + 周日 8h = 16h(相当于一周)
- 牺牲休息,但能跟上进度

**方案 B:压缩任务**
- 用户独占任务必须做(Mavis 不能代)
- Mavis 任务可以延后 1 周
- 整体进度延后 1 周

**方案 C:暂停 + 复盘**
- 如果连续 2 周进度 < 50%,暂停 1 周复盘
- 找出原因(时间不够 / 难度太大 / 失去兴趣)
- 调整计划后再启动

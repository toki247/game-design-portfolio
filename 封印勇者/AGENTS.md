## Camera 配置(2026-07-01 定)

### 视角
- **视角类型**:**45 度俯视,2:1 dimetric 投影**(塞尔达缩小帽风格,不是严格 isometric,也不是正交)
- **实现**:Godot Camera2D + Transform2D 倾斜变换(tile 32×32 → 屏幕菱形)
- Transform2D 配置:`x axis = (1, 0.5)`,`y axis = (-1, 0.5)`(让 y 方向倾斜,呈现 dimetric 风格)

### 跟随
- **跟随方式**:**固定以主角为中心**(无死区,无延迟跟随)
- **不跟随**的镜头(策划需要时单独触发):Boss 战特写、过场镜头、对话镜头

### 镜头效果(2026-07-01 用户拍板)
- **震帧(shake)**:用于编排和画面表现(命中反馈 / Boss 大招 / 关键剧情)
- **拉远拉近(zoom)**:用于编排和画面表现(过场镜头 / Boss 战特写 / 关键剧情)

### 架构决策(2026-07-01 Mavis 建议,**封装成组件,不是直接在 Camera 节点添加**)
- `CameraShake` 组件(自定义 Node 节点,挂到 Camera2D)
- `CameraZoom` 组件(自定义 Node 节点,挂到 Camera2D)
- 理由:可复用 + 策划可视化调参(@export)+ 数据驱动 + 后续多场景复用

## 尺寸规范(2026-07-01 用户拍板)

### 基础单位
- **Tile size**:32×32 像素(标准 32-bit 像素)
- **像素比**:1×(等比,无缩放)

### 主角
- **Sprite 尺寸**:32×48 像素(2 tile 宽 × 1.5 tile 高)
- **碰撞体**:16×16 像素(中心 1 个 tile)
- **移动速度**:128 像素/秒(4 tile/秒)

### 视野范围(镜头可见)
- 16×10 tile = 512×320 像素

### Boss 尺寸
- 64×64 ~ 128×128 像素(2-4 tile 大)

### 普通怪物尺寸
- 32×32 ~ 64×64 像素(1-2 tile)

### 单位约定
- 所有距离描述以"tile"或"像素"为单位
- Godot 代码里用 `TILE_SIZE = 32` 常量,避免硬编码
- 攻击范围 / 视野 / UI 位置 都基于 tile 网格

## Control 关键决策(2026-07-01)

### 按键映射

| 操作 | 按键 | 备注 |
|------|------|------|
| 移动 | WASD(八方向) | 也支持方向键 |
| 跳跃 | Space(待定) | 用户未拍板是否保留 |
| 剑 | **J** | 4 段动作(起手/挥砍/结束/收剑)+ 蓄力(强化) |
| 盾 | **U** | 防御(挡前方)+ 弹反(强化,弹反帧) |
| 翻滚 | **K** | 3 段动作(起手/翻滚/起身)+ 无敌帧 |
| 弓 | **I** | 射箭 + 落地 + 接触回收 + 蓄力(强化) |
| 护身符 | **E** | 破解黑雾 |
| 交互 | **J**(同剑) | 优先级:可交互对象 > 攻击 |

### 架构
- 按键配置:`GodotProject/scripts/game_setup.gd`(运行时 InputMap,避免手动改 .godot)
- 主角控制:`GodotProject/scripts/player.gd`
- 玩家能力数据驱动:用 `@export` 参数化,数据驱动,可视化调参
